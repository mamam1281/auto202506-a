# 프론트엔드-백엔드 구현 현황 및 개선/업데이트 가이드

---

## 1. 프론트엔드 실제 파일 구조 및 구현 상태 상세 분석

### 1.1 전체 폴더/파일 구조
- **app/**: Next.js 15+ App Router 기반, 각 게임/기능별 페이지(`slot`, `roulette`, `gacha`, `rps`, `quiz`, `profile`, `adult_content` 등)
- **components/**: 공통 UI 컴포넌트(`LoginForm`, `TokenDisplay`, `GameMenu`, `SlotMachine`, `Roulette`, `Gacha`, `EmotionFeedback` 등)
- **services/**: axios 기반 API 연동 모듈(`auth.js`, `gameApi.js`)
- **utils/**: 유틸리티 함수(`apiClient.js`, `rewardUtils.js`)
- **hooks/**: 커스텀 훅(`useEmotionFeedback.js`)
- **styles/**: Tailwind, 슬롯 애니메이션 등 CSS
- **public/**: 정적 리소스(이미지, 사운드 등)
- **__tests__/**, **cypress/**: Jest 단위테스트, Cypress E2E 테스트

### 1.2 구현된 주요 페이지/컴포넌트별 상태
| 경로 | 주요 기능 | API 연동 | 상태/비고 |
|------|----------|----------|-----------|
| app/page.jsx | 메인(로그인/토큰/게임 메뉴) | O | 정상 |
| app/slot/page.js, app/slots/page.jsx | 슬롯머신 | O | `/games/slot/play` 연동, UI/애니메이션 구현 |
| app/roulette/page.jsx | 룰렛 | △ | UI/애니메이션 구현, 실제 API 연동 미흡/불명확 |
| app/gacha/page.jsx | 가챠 | △ | UI/애니메이션 구현, 실제 API 연동 미흡/불명확 |
| app/rps/page.jsx | 가위바위보 | △ | UI/애니메이션 구현, 실제 API 연동 미흡/불명확 |
| app/profile/page.jsx | 프로필/추천/통계 | △ | 일부 더미/임시 데이터, API 연동 미흡/불명확 |
| app/adult_content/page.jsx | 성인콘텐츠 뷰어 | △ | UI만 구현, 실제 해금/콘텐츠 API 연동 미흡 |
| app/quiz/page.jsx | 퀴즈 | X | Stub(구현 미완료) |

- **공통 컴포넌트**: EmotionFeedback, Notification, TokenDisplay 등은 정상 구현 및 활용
- **테스트/유틸리티/사운드**: Jest, Cypress, 사운드 등 부가 리소스 포함

### 1.3 미구현/Stub/더미 데이터 구간
- 퀴즈(Quiz), 추천, 성인콘텐츠 등은 Stub/더미 UI, 실제 API 연동 미흡
- 일부 게임/기능은 백엔드 엔드포인트가 완전히 구현되어야 정상 동작 가능
- SlotMachine.jsx 등 일부 파일은 비어있음(중복/미사용 가능성)

## 2. 백엔드 연동/미구현 서비스 상세

### 2.1 연동된 서비스
- **슬롯머신**: `/games/slot/play` API 연동, 베팅/결과/토큰 실시간 반영
- **로그인/토큰**: `/auth/login`, `/users/me/tokens` 등 인증/토큰 API 연동

### 2.2 미구현/Stub/더미 데이터 서비스
- **퀴즈(Quiz)**: UI만 존재, 실제 문제/정답/채점/보상 등 백엔드 API 없음
- **룰렛/가챠/RPS**: UI/애니메이션은 구현, 실제 게임 결과를 백엔드에서 받아오는 API 연동이 불완전하거나 클라이언트에서만 처리
- **프로필/추천/통계**: 일부 더미/임시 데이터 사용, API 호출은 있으나 실제 백엔드 구현 여부 불명확
- **성인콘텐츠 해금/뷰어**: UI는 구현, 실제 해금/콘텐츠 목록/상세 등 API 연동 미흡

## 3. 기준문서/구조 업데이트 및 개선 가이드

### 3.1 문서/구조 업데이트 필요성
- 프론트엔드-백엔드 연동 현황을 기준문서(`01_architecture_en.md`, `04_API & Logic Flow.md` 등)에 반영
- 미구현 서비스/기능별 구체적 개선/구현 우선순위, 체크리스트 제공
- **Vite/cc-webapp-frontend 폴더/파일은 반드시 사용 금지, Next.js 15+만 공식 지원**

### 3.2 개선 우선순위 및 체크리스트
- [ ] 메인 페이지 로그인 UI, 토큰 표시, 게임 메뉴 완성
- [ ] 슬롯/룰렛/가챠/RPS/프로필/성인콘텐츠 실제 API 연동
- [ ] 퀴즈, 추천, 성인콘텐츠 등 Stub/더미 UI → 실제 백엔드 연동
- [ ] 불필요/중복/미사용 파일 정리
- [ ] 테스트(E2E, 단위) 및 커버리지 강화
- [ ] 문서화/주석/개발 가이드 최신화

### 3.3 실제 적용 예시
- **슬롯머신**: 프론트-백 완전 연동, 베팅/결과/토큰 실시간 반영, 테스트/문서화 완료
- **룰렛/가챠/RPS**: UI/애니메이션 → 실제 API 연동 및 결과/보상/통계 반영 필요
- **퀴즈/추천/성인콘텐츠**: Stub/더미 UI → 백엔드 API 설계/구현/연동 필요

## 5. 결론 및 권고

Casino-Club F2P 프로젝트의 프론트엔드-백엔드 연동 현황은 슬롯머신 등 일부 핵심 기능을 제외하면 Stub/더미 UI, 미완성 API, 불완전한 연동 구간이 다수 존재합니다. 모든 개발자는 본 가이드와 기준문서를 참고하여, 실제 서비스 수준의 연동/테스트/문서화를 달성해야 하며, Vite/cc-webapp-frontend 등 비공식 도구는 절대 사용하지 않아야 합니다. 향후 4주 내 MVP 완성 및 품질 기준 달성을 위해, 각 기능별 우선순위에 따라 집중적으로 개선/구현을 진행해야 합니다.
